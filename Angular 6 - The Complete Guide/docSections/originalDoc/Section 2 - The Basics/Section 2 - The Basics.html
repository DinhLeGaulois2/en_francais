21. Fully Understanding the Component Selector

A component is not ONLY a component but also be an "attribute" (good to create "directives"):

@Component({
   selector: '[app-own-component]',
   ...
})

//----------------------

// Usage (in a template)
// Inside a regular HTML tag, we could do, for example with a 'div':
<div app-own-component></div>

Or also be an "class":

@Component({
   selector: '.app-own-component',
   ...
})

//----------------------

// Usage (in a template)
// Inside a regular HTML tag, we could do, for example with a 'div':
<div class="app-own-component"></div>

24. Property Binding

Let say that we have a variable "allowNewServer" ("servers/servers.component.ts") which have the initial value of "false" but it will be "true" after 2 seconds:

import { Component, OnInit } from '@angular/core';

@Component({
   selector: 'app-servers',
   templateUrl: './servers.component.html',
   styleUrls: ['./servers.component.css']
})
export class ServersComponent implements OnInit {
   allowNewServer = false;

   constructor() { 
      setTimeout(() => {
         this.allowNewServer = true;
      },2000);
   }

   ngOnInit() {
   }
}

And we have ("servers/servers.component.html") a button. How could we use the allowNewServer to disable/enable the button?

Solution: by using the property binding with [ and ] like below:

<button class="btn btn-primary" [disabled]="allowNewServer">Add Server</button>

25. Property Binding vs String Interpolation

We could get the CONTENT of a property:

<p [innerText]="allowNewServer"></p>

Here, we're displaying the value of "allowNewServer" (equivalent to: <p>{{ allowNewServer }}</p>).

27. Bindable Properties and Events

How do you know to which Properties or Events of HTML Elements you may bind? You can basically bind to all Properties and Events - a good idea is to console.log() the element you're interested in to see which properties and events it offers.

Important: For events, you don't bind to onclick but only to click (=> (click)).

The MDN (Mozilla Developer Network) offers nice lists of all properties and events of the element you're interested in. Googling for YOUR_ELEMENT properties or YOUR_ELEMENT events should yield nice results.

31. Combining all Forms of Databinding

How it works?

  • Event Binding: for every character entered in the input, the function "onUpdateServerName" is called and the "serverName" is updated to the value of the input's field. So the value showed is changed (with {{ serverName }}). As the lower "input" is binded by "Two-Way-Binding" with "serverName", the value showed in its field is updated as well.
  • Two-Way-Binding: when we enter something on the lower input, the value to display (with {{ serverName }}) is updating but nothing happens to the upper input. Why? Because data is sent only when the user enters something but the "input" receives nothing from the outside.

32. Understanding Directives

Directives are Instructions in the DOM. The three types of directives in Angular 2 are:
  • attribute directives
  • structural directives
  • components (directives with template)

Example of directives:

  • ngIf
  • ng-list
  • ng-model

33. Using ngIf to Output Data Conditionally

Example:

<p *ngIf="my_condition_boolean">Show Result</p>

The result will be showed ONLY when my_condition_boolean is "true".

34. Enhancing ngIf with an Else Condition

We could create a "local reference", for example: #noServer using "ng-template" as follow:

<ng-template #noServer>
   No Server was created!
</ng-template>

Let do IF and ELSE

<p *ngIf="serverCreated; else noServer">Server was created, server name is {{ serverName }}</p>
<ng-template #noServer>     // <====== Don't forget the #
   No Server was created!
</ng-template>

36. Applying CSS Classes Dynamically with ngClass

We could activate a class (".myClass") under condition. For example, the class "onlineStyling" is activated only the value of serverStatus is 'online':

<p
   [ngStyle] = "{backgroundcolor: getColor()}"
   [ngClass]="{ onlineStyling: serverStatus === 'online'}"
>
   {{ 'Server' }} with ID {{ serverId }} is {{ getServerStatus() }}
</p>
